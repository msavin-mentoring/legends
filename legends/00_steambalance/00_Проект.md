## SteamBalance

### Проект: Платформа пополнения цифровых балансов игровых сервисов

#### Описание системы

FinTech сервис пополнения баланса игровых платформ (Steam, PUBG и других цифровых сервисов), выступающий посредником между пользователем, платёжными системами и поставщиками цифровых услуг.

Система оркестрирует транзакцию пополнения: создаёт заказ, принимает оплату через внешние платёжные шлюзы, инициирует выполнение операции у поставщика услуги и контролирует финализацию заказа. Основная задача платформы — обеспечить надёжную обработку финансовых транзакций, устойчивость к сбоям внешних систем и корректный жизненный цикл заказов.

#### Архитектура

Ключевые компоненты системы:

* Backend API сервис на PHP (Laravel) для обработки заказов и интеграций
* MySQL в качестве основной транзакционной базы данных
* Redis для очередей задач и временного хранения данных
* Асинхронные воркеры для обработки платежных webhook и интеграций
* WebSocket слой на Swoole для realtime обновления статусов и уведомлений
* Интеграции с платёжными провайдерами (карты, СБП и др.)
* Интеграции с поставщиками цифровых услуг для выполнения пополнений
* Мониторинг и наблюдаемость через Prometheus и Grafana
* Централизованное логирование и алертинг через Sentry

Взаимодействие между компонентами построено вокруг доменной модели заказа.
Обработка внешних событий (webhook платежных систем, ответы поставщиков услуг) выполняется асинхронно через очереди задач, что позволяет обеспечить идемпотентность операций и устойчивость к повторным уведомлениям.

#### Нагрузка и данные

* High-volume обработка заказов цифровых услуг
* Основная таблица заказов более 500 миллионов строк
* Высокая доля операций через внешние интеграции
* Большое количество асинхронных событий от платёжных систем
* Частые операции чтения и обновления статусов заказов

Для работы с большими объёмами данных применялись:

* keyset pagination вместо OFFSET
* оптимизированные индексы по status, created_at и id
* оптимизация запросов к таблицам заказов

#### Моя зона ответственности

Разработка backend-части системы обработки заказов и интеграций с платёжными провайдерами и поставщиками услуг.

Отвечал за:

* архитектуру жизненного цикла заказа
* обработку webhook платёжных систем
* устойчивость внешних интеграций
* оптимизацию high-load таблиц
* внедрение механизмов retry, fallback и circuit breaker
* реализацию realtime уведомлений о статусе заказов
* внедрение наблюдаемости и мониторинга системы


#### Технологии

PHP, Laravel, MySQL, Redis, Swoole, WebSocket, Docker, Prometheus, Grafana, Sentry, GitLab CI/CD.

#### Результаты

* Повышена устойчивость системы к сбоям платёжных провайдеров и внешних сервисов.
* Снижено количество зависших транзакций за счёт механизмов reconciliation и retry.
* Кратно ускорены выборки из таблицы заказов благодаря оптимизации индексов и переходу на keyset pagination.
* Снижена нагрузка на backend API за счёт realtime уведомлений через WebSocket вместо polling.
* Улучшена наблюдаемость системы и сокращено время реакции на инциденты.
