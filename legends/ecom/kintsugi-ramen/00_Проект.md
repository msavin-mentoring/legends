### Проект: Платформа self-service заказов для сети ресторанов Panda

#### Описание системы

Система цифрового меню и оформления заказов для сети ресторанов восточной кухни **Panda**. На каждом столе размещались планшеты, через которые гости могли просматривать меню, оформлять заказ, оплачивать его и вызывать официанта без участия персонала.

Каталог блюд, категории, теги и складские остатки синхронизировались с облачной POS-системой **iiko**. Гость выбирал блюда на планшете, формировал корзину и инициировал оплату через **Систему быстрых платежей (QR-код)**. После подтверждения оплаты заказ отправлялся в iiko для дальнейшей обработки на кухне и в системе учета ресторана.

Система также поддерживала вызов официанта: планшет отправлял запрос в отдельный микросервис на Python, который взаимодействовал с приемником **RCall** и рассылал радиосигнал на пейджеры персонала.

В архитектуре присутствовали несколько независимых систем: планшетное приложение, backend сервиса меню, POS-система iiko, платежный провайдер и сервис вызова официантов. Основной сложностью проекта была **синхронизация состояний между системами**: заказ мог быть оплачен, но не зафиксирован в POS, либо позиции могли дублироваться при передаче. Для устранения рассинхронизации реализовывались блокировки, reconciliation-процедуры и аудит действий.

Доменная модель включала:

* категории меню
* блюда
* динамические свойства блюд (product → property → value)
* столы
* заказы
* платежи
* вызовы официантов

После передачи заказа в POS-систему он удалялся из основной таблицы заказов и сохранялся только в аудите.

#### Архитектура

**Backend сервис**

* API для планшетов: каталог меню, корзина, оформление заказа, оплата, вызов официанта
* синхронизация каталога и складских остатков с iiko Cloud
* reconciliation-джобы для поиска расхождений между системами
* аудит действий пользователей и интеграций

**База данных**

* PostgreSQL
* таблицы каталога (products, categories, tags)
* EAV-модель для динамических свойств блюд
* таблицы заказов
* журнал аудита

**Кеш**

* Redis для кеширования каталога меню и изображений (меню редко изменялось)

**Интеграции**

* iiko Cloud — каталог товаров, остатки, регистрация заказов
* платежный API Тинькофф — оплата через QR СБП
* Python-микросервис вызова официанта
* радиопейджинговая система RCall

**Взаимодействие компонентов**

```
планшет
   ↓
backend (Laravel API)
   ↓
iiko Cloud / платежный API
   ↓
кухня и POS терминал
```

Для вызова официанта:

```
планшет
   ↓
backend
   ↓
python сервис
   ↓
rcall приемник
   ↓
радиопейджеры официантов
```

#### Нагрузка и данные

* сеть ресторанов: 3 точки
* количество столов: ~20–30 на ресторан
* каталог меню: ~20 категорий, ~200 позиций
* пользователи: до нескольких десятков активных планшетов одновременно
* заказы после передачи в POS удалялись из основной таблицы и сохранялись в журнале аудита

#### Моя зона ответственности

* разработка backend API системы цифрового меню
* интеграция с iiko Cloud для синхронизации каталога, остатков и заказов
* интеграция с платежным API для оплаты через СБП QR
* разработка механизма reconciliation для устранения рассинхронизации заказов между системой планшетов и POS
* реализация блокировок и идемпотентности при создании заказов
* проектирование EAV-модели для хранения динамических свойств блюд
* реализация очистки столов при длительной неактивности
* разработка подробного аудита действий для диагностики ошибок интеграций
* участие в расследовании проблем с системой вызова официантов и интеграцией RCall

#### Технологии

PHP, Laravel 10, PostgreSQL, Redis, Python, REST API, Docker, виртуальные машины
