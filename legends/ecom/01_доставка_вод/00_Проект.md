### Проект: Платформа автоматизации доставки питьевой воды

#### Описание системы

Серверная часть экосистемы доставки питьевой воды и сопутствующих товаров (B2C e-commerce), включающая два ключевых пользовательских контура: **мобильное приложение клиентов для оформления заказов** и **мобильное приложение экспедиторов для исполнения доставки**.

Клиентское приложение позволяло пользователям оформлять заказы на доставку воды и сопутствующих товаров, управлять адресами доставки, выбирать временные интервалы, применять акции и оплачивать заказы. Backend обеспечивал обработку заказов, управление каталогом товаров, хранение клиентских данных, интеграцию с платежными сервисами и передачу заказов во внутренние операционные системы компании.

Второй контур системы использовался сотрудниками компании — экспедиторами и операторами. Через мобильное приложение они получали маршруты доставки, подтверждали выполнение заказов, фиксировали складские операции и сдачу выручки. Backend координировал работу заказов, складских остатков, маршрутов и производственных процедур.

Система также поддерживала предрейсовые процессы, обязательные для сотрудников перед началом смены: проведение **медицинского осмотра**, **технического осмотра транспорта**, регистрацию погрузки продукции и подготовку маршрутов доставки.

Отдельный важный сценарий системы — утренний критичный импорт операционных данных из ERP-системы предприятия.
ERP ежедневно перед началом смены выгружала на backend крупный JSON-пакет с заказами на текущий день и связанными сущностями: медосмотры, техосмотры, погрузки и другой производственной информацией. От корректности и скорости обработки этого пакета зависел старт рабочего дня экспедиторов и доступность актуальных заказов в мобильном приложении.

Таким образом система выступала центральным backend-контуром, синхронизирующим данные между **клиентскими приложениями, внутренними операционными сервисами и ERP системой предприятия**.

#### Архитектура

Ключевые компоненты системы:

* Backend API сервис на PHP (Laravel) для клиентского и экспедиторского мобильных приложений
* PostgreSQL в качестве основной транзакционной базы данных
* Redis для кэширования данных и ускорения часто используемых операций
* RabbitMQ для асинхронной обработки задач и ретраев интеграций
* Интеграция с ERP системой предприятия (1С) через REST API
* Интеграция с Yandex Suggestions API для нормализации адресов
* Мониторинг метрик и состояния системы через Prometheus и Grafana

Взаимодействие построено по принципу REST API между мобильными клиентами и backend сервисом. Интеграции с внешними системами и обработка ошибок вынесены в асинхронные очереди RabbitMQ для повышения отказоустойчивости.

#### Нагрузка и данные

* Более 2500 активных пользователей системы
* Высокая конкурентная нагрузка на операции оформления и обработки заказов
* Асинхронные очереди RabbitMQ для обработки задач интеграции и повторных попыток
* Основные доменные сущности: заказы, клиенты, адреса доставки, складские остатки, акции и тарифы
