# 2. Проект

## HR-система
Система хранит информацию о всех сотрудниках группы компаний: персональные данные, должность и  
место работы, заработная плата и иные выплаты, информацию о тестировании, образование и др.  
Также проект является master-системой для получения и актуализации данных о сотрудниках другими  
системами в контуре группы компаний.  

Функционал системы:  
- Создание и редактирование компаний и их подразделений
- Создание и редактирование карточки сотрудника
- Редактирование руководителем компетенций сотрудника  
  (В ГК используется система компетенций и грейдов от которых зависит зарплата)
- Просмотр инфографики по тестированию персонала (интеграция с системой тестирования)
- Просмотр оценки руководителя по итогам аудита (интеграция с системой проведения аудитов)
- Выгрузка отчетов по пройденным тестам (процент пройденных тестов, оценки по тестам и т.д.)
- Выгрузка отчетов по сотрудникам подразделения (должности, грейды и т.д.)
- Просмотр инфографики по заработным платам (интеграция с 1с ЗУП)

## Система учета Патентов
В системе хранится информация об объектах интеллектуальной собственности (товарные знаки,  
полезные модели, патенты и т.д.) зарегистрированные компаниями входящими в ГК. Патенты закреплены  
за отдельными компаниями, имеют срок действия, описание, ответственных юристов, размер госпошлины  
и сроки оплаты пошлины.

Функционал системы:
- Создание и редактирование объектов индивидуальной собственности
- Автоматическая актуализация данных по патентам с Роспатентом (через апи организации ФИПС)
- Система уведомлений для юристов, которая оповещает о патентах с заканчивающимся сроком действия
- Автоматическое формирование платежных поручений для оплаты госпошлины (интеграция с 1с)

## Система тестирования персонала
Сотрудники компании проходят психологические тесты на основе которых формируется психологический  
портрет человека (на основе этого портрета подбирается команда, чтобы тихони работали с тихонями и т.д.).  
Данная информация была доступна в различных отчетах и передавалась в другие системы  
(например HR-систему и др.). Также в этой системе проводятся аттестации для проверки уровня  
профессиональных навыков.

Функционал системы:
- Конструктор для создания тестов (добавление картинок, разные варианта ввода ответа,  
  добавление таймера)
- Выгрузка отчетов по пройденным тестам 
- Система уведомлений для сотрудников, которым необходимо пройти тестирование

## Система для проведения аудита предприятий
Система для хранения данных по проведенным аудитам предприятий. Предприятие разделяется по отделам,  
за которыми закрепляются ответственные руководители. В течение месяца проводятся проверки.  
Несовершенства (сломанное оборудование, грязный цех и т.д.) заносятся в систему, закрепляются  
ответственные по устранению несовершенств и сроки для их устранения. После устранения проблемы  
фиксируется время и качество работы (путем опроса того, кто завел несовершенство в систему),  
которые влияют на конечную оценку эффективности руководителя. Раз в месяц рассчитывается рейтинг,  
в котором учитываются все активные в этом месяце несовершенства. На основе этих рейтингов рассчитывается  
размер премии руководителей.

Функционал системы:
- Создание и редактировании структуры предприятия, назначение ответственных (интеграция с HR-системой)
- Механизм создания и редактирования несовершенств с возможностью прикреплять фото и видео
- Личный кабинет руководителя, для просмотра несовершенств его отдела и дочерних отделов
- Система оповещения руководителей о новых заявках и заявках, у которых истекает срок на исполнение
- Расчет рейтинга руководителй на основе эффективности решения проблем отдела
- Выгрузка отчетов по устранению несовершенств и рейтингов

## Система управления инцидентами (аналог jira)
Система для трекинга рабочего времени, обработки заявок в разные ит-отделы, создание и описание задач,  
ведение проектов, страница согласования отпусков сотрудников. Большое количество функционала реализована,  
реализовано аналогично jira.  

Функционал системы:
- Трекинг рабочего времени в режиме реального времени
- Создание заявок, описание их и оценка (как в jira)
- Создание проектов (аналог эпика в jira) с функционалом согласования руководителями
- Механизм согласования отпусков сотрудников (интерактивный календарь)
- Механизм оформления отсутствий на работе (больничные, выходные без содержания и т.д.)

## Система для проведения комитетов
Система для проведения видеоконференций сотрудников.

Функционал системы:
- Календарь для бронирования времени на созвоны
- Сотрудники и подразделения подтягиваются из HR-системы (интеграция с HR-системой)
- Функционал видеозвонков (интеграция с Big Blue Button)

## Система учета ресторана
На закрытом производственном комплексе есть ресторан для сотрудников. Система позволяет бронировать  
места в ресторане и совершать предзаказ блюд по меню.

Функционал системы:
- Редактирование схемы зала ресторана (система тиражировалась на несколько ресторанов)
- Добавление блюд с указанием состава и т.д.
- Формирование меню ресторана на неделю
- Механизм бронирование столика
- Механизм предоплаты заказа
- Хранение информации о заказах (позиции меню, сотрудник-заказчик, стоимость и т.д.) (интеграция с 1с)

# 3. Бизнес-поток

В команде работают по scrum с недельными спринтами  
Разработчики проводят кросс-ревью  
Ветвление в git по gitflow

1. Заказчик (представитель от бизнеса) заводит заявку в системе управления инцидентами
2. Старший бизнес-аналитик прорабатывает заявку
3. В конце спринта ПМ с аналитиками формируют бэклог спринта
4. В течении спринта разработчик берет задачи из бэклога, уточняет требования по задаче у аналитика 
5. Разработчик проходит архитектурную постановку у архитектора системы
6. Разработчик выполняет доработку
7. Другой разработчик проводит кросс-ревью
8. Архитектор системы проводит ревью
9. Задача тестируется аналитиком
10. Задача попадает в ближайший релиз
