### Внутренняя система учета патентов

Система предназначена для хранения и управления данными об объектах интеллектуальной собственности, таких как  
товарные знаки, полезные модели, патенты и другие подобные объекты, зарегистрированные компаниями, входящими  
в группу компаний (ГК). В системе фиксируется информация о принадлежности патентов конкретным компаниям,  
их сроках действия, описаниях объектов, ответственных юристах, размере государственной пошлины и сроках её оплаты.

Эта система служит основным источником данных о правах на интеллектуальную собственность в группе компаний  
и используется другими информационными системами ГК для получения, актуализации и управления 
соответствующей информацией.

Основной функционал:
- Управление объектами интеллектуальной собственности: создание и редактирование патентов, товарных знаков,  
полезных моделей и других объектов
- Автоматическая синхронизация данных по патентам с Роспатентом через API ФИПС
(ежедневный обмен по cron, https://www1.fips.ru)
- Система уведомлений для юристов о патентах с истекающим сроком действия
- Автоматическое формирование платежных поручений для оплаты государственной пошлины 
(интеграция с 1С)
- Конструктор правил для определения срока действия патента и расчёта суммы государственной пошлины
- Аналитика и отчётность по патентам: создание отчетов о состоянии портфеля интеллектуальной собственности,  
анализ сроков действия, объема госпошлин и других ключевых показателей

#### Отвечал за:
- Backend-разработку системы управления данными об объектах интеллектуальной собственности
- Настройку и поддержку системы уведомлений для юристов о патентах с истекающим сроком действия
- Разработку системы отчетности и аналитики по патентам, включая создание отчетов о состоянии портфеля 
интеллектуальной собственности, анализ сроков действия и объема госпошлин
- Интеграции с внешними системами документооборота

#### Команда:
- 3 backend-разработчиков
- 1 frontend-разработчик
- 1 project-менеджер
- 2 бизнес-аналитика

#### Флоу работы:
- Стейкхолдер создает заявку на доработку
- Задачу прорабатывает бизнес-аналитик
- Бизнес-аналитик передает задачу разработчику
- Разработчик описывает решение и проходит архитектурную постановку у архитектора системы
  (архитектор - главный разработчик системы)
- Разработчик выполняет доработку
- Бизнес-аналитик тестирует доработку на тестовом инстансе приложения
- Другой разработчик проводит код ревью
  (в рамках кросс-ревью)
- Архитектор проводит код ревью
- Задача попадает в ближайший релиз системы
- После релиза автор задачи проверяет доработку
- Задача закрывается

#### Достижения:
- Спроектировал интеграцию с Роспатентом через API ФИПС, что обеспечило синхронизацию данных о патентах  
и полностью автоматизировало процесс актуализации сведений о них
- Спроектировал систему уведомлений для юристов, что позволило своевременно отслеживать патенты с истекающим сроком  
действия и снизить риски утраты прав на интеллектуальную собственность
- Спроектировал автоматизацию формирования платежных поручений для оплаты государственной пошлины, интегрировав  
систему с 1С, что ускорило финансовые процессы и уменьшило вероятность ошибок
- Спроектировал конструктор правил для расчёта сроков действия патентов и госпошлин, что позволило оперативно  
адаптировать систему к изменениям законодательства без вмешательства разработчиков
- Спроектировал механизм автоматической передачи объектов интеллектуальной собственности между юридическими лицами,  
включающий пересчет государственной пошлины и автоматическую подготовку заявления в Роспатент для оформления изменений

#### Технологии:
PHP, Symfony, PostgreSQL, Docker, Redis, RabbitMQ, Keycloak, Swagger
