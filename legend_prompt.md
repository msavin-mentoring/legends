## PROMPT ДЛЯ ГЕНЕРАЦИИ ЛЕГЕНДЫ ПРОЕКТА

Сформируй профессиональную проектную легенду для резюме senior backend-разработчика на основе предоставленных данных о проекте.

Требования:

1. Строго соблюдай структуру разделов.
2. Ничего не выдумывай, если нужной информации нет - задавай уточняющие вопросы
3. Пиши в деловом, техническом стиле.
4. Используй конкретику: метрики, нагрузки, SLA, объёмы данных, RPS.
5. Подчёркивай бизнес-ценность.
6. Формулируй так, чтобы текст подходил для интервью и можно было раскрывать детали.
7. Не сокращай технические термины.
8. Не добавляй лишних разделов.
9. Не меняй порядок блоков.
10. Если достижения уже описаны - используй их, не меняй

## СТРУКТУРА ВЫВОДА

```

Что это за продукт и какую задачу бизнеса решает.

#### Архитектура
Опиши ключевые компоненты системы:

* backend сервисы
* базы данных
* очереди и брокеры сообщений
* кеши

* взаимодействие между компонентами

* RPS
* количество пользователей

* размеры ключевых таблиц

#### Моя зона ответственности

Ключевые архитектурные решения.

#### Технологии
PHP, Symfony, PostgreSQL, Redis, RabbitMQ, Docker …
```
