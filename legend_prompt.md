## PROMPT ДЛЯ ГЕНЕРАЦИИ ЛЕГЕНДЫ ПРОЕКТА

Сформируй профессиональную проектную легенду для резюме senior backend-разработчика на основе предоставленных данных о проекте.

Требования:

1. Строго соблюдай структуру разделов.
2. Ничего не выдумывай, если информации нет — аккуратно обобщай.
3. Пиши в деловом, техническом стиле.
4. Используй конкретику: метрики, нагрузки, SLA, объёмы данных, RPS.
5. Подчёркивай бизнес-ценность.
6. Формулируй так, чтобы текст подходил для интервью и можно было раскрывать детали.
7. Не сокращай технические термины.
8. Если данных мало — логично достраивай реалистичные детали уровня enterprise.
9. Не добавляй лишних разделов.
10. Не меняй порядок блоков.

## ДОПОЛНИТЕЛЬНЫЕ ПРАВИЛА

* Если есть Kafka — опиши события, идемпотентность, partitioning.
* Если есть PostgreSQL — упомяни индексы, партиционирование, оптимизацию.
* Если есть Redis — опиши стратегию кеширования.
* Если есть SLA — обязательно укажи процент и как он достигался.
* Если есть нагрузка — покажи, что архитектура выдерживала пиковые сценарии.
* Метрики должны выглядеть правдоподобно (±10–20% допустимо, но без фантазий в 10x).
* Формулировки должны быть такими, чтобы их можно было защищать на техническом интервью.
